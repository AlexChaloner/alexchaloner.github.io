Start with the problem

How do we turn random noise into new data?

Suppose the orange points are examples from a dataset. We can draw more Gaussian noise whenever we want, but we cannot directly draw another data point. A generative model has to learn a transformation from that easy source of randomness into the shape of the data.

Diffusion gives us one answer: gradually add noise to data, train a model to identify the noise that was added, then run that corruption process backward.

Change the learning target

Flow matching learns the trip, not the corruption

Flow matching starts with the same ingredients—data, Gaussian noise, and a randomly chosen intermediate time—but constructs an explicit path from noise to data. The model’s target is now the path’s velocity: which way and how fast should this point move right now?

xt = (1 − t) xnoise + t xdata ut = xdata − xnoise min E ‖vθ(xt, t) − ut‖²
DiffusionLearn the noise or scoreReverse a noising processSample with a reverse SDE or its probability-flow ODE
Flow matchingLearn velocity directlyFollow a chosen probability pathSample by solving an ODE

The boundary is not absolute. Flow matching is a general way to train vector fields, and it can use diffusion probability paths too. This toy compares the familiar noise-prediction diffusion objective with the simplest straight-line, independently paired flow-matching objective.

Why are the connections arbitrary?

The pairs are temporary scaffolding—not destinations chosen by particles

  1. Independently draw one Gaussian-noise sample and one example from the dataset.
  2. Temporarily connect them so the velocity along that path is known.
  3. Train on current position + time → velocity. Do not show the model the endpoint.
  4. Discard the pair. At generation time, fresh noise follows the averaged field and has no assigned destination.

So there is no hidden rhyme or reason to any one line. The structure appears only after the model averages the enormous collection of random training paths. It is not trained by running the eight modes backward: training jumps directly to a random intermediate time; only generation integrates a journey, from noise to data.

The important subtlety

One location can receive conflicting instructions

Random paths cross. Near a crossing, one pair may say “up” while another says “down”. A deterministic model cannot emit both answers, so squared-error regression learns their conditional average. That average is the marginal velocity field: the single vector field whose probability flow has the same intermediate distributions as all those conditional paths.

The second view lets you click anywhere to inspect this averaging. Thin arrows are nearby training targets; the heavy arrow is their weighted mean.

What to keep

The flow matching mental model

  1. Create easy supervised targets. Sample noise, sample data, choose a probability path, and calculate its velocity.
  2. Regress many conditional arrows into one field. The model learns the average instruction compatible with its current state and time.
  3. Generate by solving an ODE. Start from new noise and repeatedly ask the field which way to move.

Real systems replace the two-dimensional points with images, audio, actions, or other high-dimensional objects. The geometry is harder to draw, but the recipe is unchanged.

Can both methods learn handwritten digits?

Real MNIST digits, downsampled to 8 × 8 pixels so two small generative models can begin from scratch and train live on identical batches.

Nothing here is pretrained. Each model works directly on 64 pixel values. The lower resolution keeps live CPU training practical; both randomly initialized fields learn from 6,000 MNIST examples. In the all-digits view, each model is also told the requested digit class—the standard class-conditioned setup—while still learning every pixel from scratch.
